§ dbtree_fetch()

int dbtree_fetch ( daos_handle_t  toh,
dbtree_probe_opc_t  opc,
daos_iov_t *  key,
daos_iov_t *  key_out,
daos_iov_t *  val_out 
)

Search the provided key and fetch its value (and key if the matched key is different with the input key). This function can support advanced range search operation based on opc.

If key_out and val_out provide sink buffers, then key and value will be copied into them. Otherwise if buffer address in key_out or/and val_out is/are NULL, then addresses of key or/and value of the current record will be returned.

Parameters
toh[IN] Tree open handle.
opc[IN] Probe opcode, see dbtree_probe_opc_t for the details.
key[IN] Key to search
key_out[OUT] Return the actual matched key if opc is not BTR_PROBE_EQ.
val_out[OUT] Returned value address, or sink buffer to store returned value.
Returns
0 found -ve error code

§ dbtree_iter_probe()

int dbtree_iter_probe ( daos_handle_t  ih,
dbtree_probe_opc_t  opc,
daos_iov_t *  key,
daos_hash_out_t *  anchor 
)

Based on the opc, this function can do various things:

  • set the cursor of the iterator to the first or the last record.
  • find the record for the provided key.
  • find the first record whose key is greater than or equal to the key.
  • find the first record whose key is less than or equal to the key.

This function must be called after dbtree_iter_prepare, it can be called for arbitrary times for the same iterator.

Parameters
opc[IN] Probe opcode, see dbtree_probe_opc_t for the details.
key[IN] The key to probe, it will be ignored if opc is BTR_PROBE_FIRST or BTR_PROBE_LAST. Otherwise, it is required if BTR_FEAT_DIRECT_KEY is used.
anchor[IN] the anchor point to probe, it will be ignored if key is provided.
