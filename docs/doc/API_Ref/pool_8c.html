)

Retrieve a list of user-defined pool attribute values.

Parameters
poh[IN] Pool handle
n[IN] Number of attributes
names[IN] Array of n null-terminated attribute names.
buffers[OUT] Array of n buffers to store attribute values. Attribute values larger than corresponding buffer sizes will be truncated. NULL values are permitted and will be treated identical to zero-length buffers, in which case only the sizes of attribute values will be retrieved.
sizes[IN] Array of n buffer sizes. [OUT] Array of actual sizes of n attribute values, regardless of given buffer sizes.
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.

List the names of all user-defined pool attributes.

Parameters

daos_pool_info_t *  info,

Connect to the DAOS pool identified by UUID uuid. Upon a successful completion, poh returns the pool handle, and info returns the latest pool information.

Parameters
uuid[IN] UUID to identify a pool.
grp[IN] Process set name of the DAOS servers managing the pool
svc[IN] Pool service replica ranks, as reported by daos_pool_create().
flags[IN] Connect mode represented by the DAOS_PC_ bits.
poh[OUT] Returned open handle.
info[OUT] Returned pool info.
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.
Returns
These values will be returned by ev::ev_error in non-blocking mode: 0 Success -DER_INVAL Invalid parameter -DER_UNREACH Network is unreachable -DER_NO_PERM Permission denied -DER_NONEXIST Pool is nonexistent

§ daos_pool_local2global()

int daos_pool_local2global ( daos_handle_t  poh,
daos_iov_t *  glob 
)

Handle API Convert a local pool connection to global representation data which can be shared with peer processes. If glob->iov_buf is set to NULL, the actual size of the global handle is returned through glob->iov_buf_len. This function does not involve any communication and does not block.

Parameters
poh[IN] Valid local pool connection handle to be shared
glob[OUT] Pointer to iov of the buffer to store handle information
Returns
These values will be returned: non-blocking mode: 0 Success -DER_INVAL Invalid parameter -DER_NO_HDL Pool handle is nonexistent -DER_TRUNC Buffer in glob is too short, a larger buffer is required. In this case the required buffer size is returned through glob->iov_buf_len.
