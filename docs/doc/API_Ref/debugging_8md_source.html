1 # DAOS Debugging
2 
3 D_LOG_FILE:
4  DAOS debug logs are written by default to /tmp/daos.log, this can be
5  modified by specifying a different path through D_LOG_FILE.
6 
7 D_LOG_MASK:
8  Used to set logging mask for either all subsystems (ie D_LOG_MASK=INFO
9  will set all subsystems log level to INFO) or specific subsystems
10  (ie D_LOG_MASK=ERR,RPC=DEBUG,BULK=INFO).
11 
12 DD_MASK:
13  Can enable different debug streams for finer-grained debugging per
14  subsystem, for example "DD_MASK=trace" will log only debug messages
15  with the bit mask DB_TRACE associated with them. Multiple streams can be
16  set as well, for example: "DD_MASK=io,mem" will allow logging of a
17  subset of debug messages related to io path (bit mask=DB_IO) and memory
18  operations (bit mask=DB_MEM). Since these are debug masks, the
19  subsystem/facility must have the debug log set ("D_LOG_MASK=DEBUG"),
20  otherwise other priorities take precedence. The bit streams are passed
21  as a parameter to D_DEBUG(mask, fmt, ...).
22 
23  Current DAOS debug masks:
24  md/metadata, pl/placement, mgmt/management, epc/epoch,
25  df/durafmt, rebuild
26  Current GURT/common debug masks:
27  any, all, mem, net, io, trace, test
28 
29 DD_STDERR:
30  User can specify the priority level to output to stderr, for example:
31  "DD_STDERR=info" will log all DLOG_INFO priority messages to stderr.
32 
33 DD_SUBSYS:
34  User can specify which subsystems to enable.
35  DAOS facilites disabled by default:
36  common, tree, vos, utils, tests
37  DAOS facilites enabled by default:
38  null, client, server, rdb, pool, container, object, placement,
39  rebuild, tier, mgmt
40  DD_SUBSYS=all will enable all facilities.
