See the License for the specific language governing permissions and limitations under the License. DAOS Management API. Create a pool spanning tgts in grp. Upon successful completion, report back the pool UUID in uuid and the pool service rank(s) in svc, which are required by daos_pool_connect() to establish a pool connection.

Targets are assumed to share the same dev and size.

Parameters
mode[IN] Capabilities permitted for the pool. May contain these bits: 0400 user DAOS_PC_EX 0200 user DAOS_PC_RW 0100 user DAOS_PC_RO 0040 group DAOS_PC_EX 0020 group DAOS_PC_RW 0010 group DAOS_PC_RO 0004 other DAOS_PC_EX 0002 other DAOS_PC_RW 0001 other DAOS_PC_RO
uid[IN] User owning the pool
gid[IN] Group owning the pool
grp[IN] Process set name of the DAOS servers managing the pool
tgts[IN] Optional, allocate targets on this list of ranks If set to NULL, create the pool over all the ranks available in the service group.
dev[IN] String identifying the target devices to use
size[IN] Target sizes in bytes (i.e., maximum amounts of storage space targets can consume) in bytes. Passing 0 will use the minimal supported target size.
svc[IN] Number of desired pool service replicas. Callers must speicfy svc->rl_nr and allocate a matching svc->rl_ranks; svc->rl_nr and svc->rl_ranks content are ignored. [OUT] List of actual pool service replicas. svc->rl_nr is the number of actual pool service replicas, which shall be equal to or smaller than the desired number. The first svc->rl_nr elements of svc->rl_ranks shall be the list of pool service ranks.
uuid[OUT] UUID of the pool created
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.

int daos_pool_destroy ( const uuid_t  uuid,
const char *  grp,
int  force,
daos_event_t *  ev 
)

Destroy a pool with uuid. If there is at least one connection to this pool, and force is zero, then this operation completes with DER_BUSY. Otherwise, the pool is destroyed when the operation completes.

Parameters
uuid[IN] UUID of the pool to destroy
grp[IN] Process set name of the DAOS servers managing the pool
force[IN] Force destruction even if there are active connections
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.

Exclude completely a set of storage targets from a pool. Compared with daos_pool_exclude(), this API will mark the targets to be DOWNOUT, i.e. the rebuilding for this target is done, while daos_pool_exclude() only mark the target to be DOWN, i.e. the rebuilding might not finished yet.

Parameters
uuid[IN] UUID of the pool
grp[IN] process set name of the DAOS servers managing the pool
svc[IN] list of pool service ranks
tgts[IN] Target rank array to be excluded from the pool. Now can-only exclude one target per API calling.
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.
Returns
These values will be returned by ev::ev_error in non-blocking mode: 0 Success -DER_INVAL Invalid parameter -DER_UNREACH Network is unreachable -DER_NO_PERM Permission denied

§ daos_pool_extend()

int daos_pool_extend ( const uuid_t  uuid,
const char *  grp,
d_rank_list_t *  tgts,
d_rank_list_t *  failed,
daos_event_t *  ev 
)

Extend the pool to more targets. If tgts is NULL, this function will extend the pool to all the targets in the group, otherwise it will only extend the pool to the included targets.

NB: Doubling storage targets in the pool can have better performance than arbitrary targets adding.

Parameters
uuid[IN] UUID of the pool to extend
grp[IN] Process set name of the DAOS servers managing the pool
tgts[IN] Optional, only extend the pool to included targets.
failed[OUT] Optional, buffer to store faulty targets on failure.
ev[IN] Completion event, it is optional and can be NULL. The function will run in blocking mode if ev is NULL.
Returns
These values will be returned by ev::ev_error in non-blocking mode: 0 Success -DER_INVAL Invalid parameter -DER_UNREACH Network is unreachable -DER_NO_PERM Permission denied -DER_NONEXIST Storage target is nonexistent
