Each replica employs four daemon ULTs:

~ rdb_timerd(): Call raft_periodic() periodically. ~ rdb_applyd(): Apply committed entries. ~ rdb_recvd(): Process RPC replies received. ~ rdb_callbackd(): Invoke user dc_step_{up,down} callbacks.

rdb maintains its own last applied index persistently, instead of using raft's volatile version.

rdb's raft callbacks may return rdb errors (e.g., -DER_IO, -DER_NOSPACE, etc.), rdb's and raft's error domains are disjoint (see the compile-time assertion in rdb_raft_rc()).
